## On the project

The core bet — that an honest work-item tracker's identity/auth layer *is* the
root of provenance, and its activity log *is* the verified chain — holds up
beautifully when you build it. regista's facade is clean to consume, and the
"gateway as sole choke point + server-resolved Actor" discipline makes G1
(attribution) a *structural* property rather than a convention: there is
literally no method on `RegistaGateway` that accepts an `actor_id` string, so
spoofing the actor requires changing the type signature, not exploiting a bug.
That's the right shape.

The design spine (`docs/provenance-model.md`) earned its keep: every time I
had a decision (does the creator count as an author? where do DOSSIER-N keys
live?), the doc either answered it or correctly flagged it as a
human-decision. The plans (001–005) sequenced cleanly. The one place the
charter was thin: the workflow had no `title`/`description` fields — a tracker
needs them, and adding them is a contract change I had to surface rather than
slip in.

## Gaps to flag

- **WI-004 / validator delegation gap** (`src/dossier/validators.py`): the
  reviewer's own `on_behalf_of` isn't on regista's `ValidatorContext`, so an
  agent reviewing on behalf of an author isn't caught as self-review. Not
  exploitable in the all-human MVP; latent for the mixed human+agent north
  star. Needs regista Plan 021.
- **WI-003 / integrity badge** (`src/dossier/templates/_history.html`): uses
  `gateway.integrity()` which replays the *whole project* — drift anywhere
  surfaces on every issue's badge. Needs a per-work-item replay API in
  regista for a precise per-item status.
- **No visual confirmation of the UI**: I verified structure/text, not
  appearance. Someone with image input should look at the plum/patina render.
- **`current_actor_or_redirect`** (`src/dossier/app.py:87`): uses
  `HTTPException(302)` which leaves a JSON body on redirect (WI-001).
  Functional, ugly.
- **Workflow v2** (`src/dossier/workflows/dossier.workflow.yaml`): re-running
  `dossier init` is required for any dev project that registered v1; a
  content change on a registered version raises `WORKFLOW_VERSION_CONFLICT`.
- **CSRF is per-route** (`Depends(verify_csrf)`), not enforced by middleware —
  a future POST route that forgets the dependency is unprotected (WI-007).
  Add a conformance test or middleware.
