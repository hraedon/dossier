## On the project

The core suite-cohesion features are now real: config adoption (`REGISTA_*` canonical + `DOSSIER_*` deprecated aliases), `doctor --json`, `/healthz`, container image with pinned regista, idempotent first-run with provision check, and now `suite.env` loading. The gap I found — suite.env wasn't actually implemented despite being in the plan — is the kind of thing that happens when a plan's WI-1.1 says "resolve via suite precedence" and the implementation only handles the env-var aliasing, not the file loading. The blueprint is detailed enough that this was easy to spot.

The multi-stage Dockerfile fix was straightforward but the REGISTA_REF mismatch was a real latent bug: the Dockerfile had `3613d95...` while `SUITE.lock` had `d7d156c...`. The CI job that checks pin-matching would have caught it, but only on the next build. The venv-copy approach (instead of `--prefix`) works cleanly and produces a smaller runtime layer.

## On the work done

The suite.env loader is the meat of this session. I'm confident in the hardened version: `O_NOFOLLOW` prevents symlink attacks, world-writable files are rejected, dangerous keys (`PYTHONPATH`, `LD_PRELOAD`) are blocked, BOM is handled, and the idempotency flag prevents double-loading. The `_strip_value` function correctly handles the quoted-value-with-space-hash case that both reviewers flagged — this was the most subtle bug.

Moving `_configure_admin_ids()` from module-import to `create_app()` was the right structural fix. The module-import side effect meant admin IDs from suite.env would be invisible in any non-CLI invocation path (uvicorn direct, tests). Now it runs inside `create_app()` where suite.env has already been loaded by the CLI.

What I'm less confident in: the `_strip_value` quote-matching logic uses `raw.rfind(raw[0])` to find the closing quote, which means a value like `"it's"` would return `it's` correctly, but `"a"b"c"` would return `a"b` (finding the last `"`). This is an edge case unlikely to appear in real `.env` files, but it's not technically correct shell quoting. I'm accepting this as a v1 simplification.

## Gaps to flag

- `src/dossier/config.py:_strip_value` — Quote matching uses `rfind` which is imperfect for values with embedded quotes (`"a"b"`). Acceptable for `.env` files but not shell-correct.
- `Dockerfile:42` — Base image `python:${PYTHON_VERSION}-slim` is not digest-pinned; successive pulls could resolve to different minor versions and cause ABI mismatch between builder and runtime stages.
- `docker-compose.yml` — Still references `REGISTA_KEY_PATH` in the environment but doesn't mount a `keys.json` volume by default; a fresh `docker compose up` would fail at runtime with no key file. The `volumes` section maps `./keys.json` which must exist locally.
- `src/dossier/app.py:83` — `_configure_admin_ids()` is now called in `create_app()` but the module-level `_ADMIN_ACTOR_IDS: set[str] = set()` is still a module-level mutable. Multiple `create_app()` calls in the same process (e.g., tests) would accumulate/clear admin IDs across instances. Not a real problem today (single-app processes) but worth noting.
- The `_SUITE_ENV_LOADED` module-level flag in `config.py` means tests that need to re-load suite.env must reset it manually (the test file does this via a fixture, but it's a gotcha for future test authors).
