# dossier Windows Service deployment (Plan 013 WI-4.1)

## Overview

dossier runs on Windows as a service via [WinSW](https://github.com/winsw/winsw),
the wrapper validated in the Plan 016 spike. WinSW is a single 18MB exe + XML
config — no Python service code needed (pywin32's service framework is broken
on Python 3.14).

## Prerequisites

- **Python 3.12+** installed and on PATH
- **WinSW v2.12.0+** — download `WinSW-x64.exe` from
  [GitHub releases](https://github.com/winsw/winsw/releases)
- **Postgres** reachable from the Windows host
- **regista provision** already run for the target project

## Install steps

1. **Download WinSW** and rename it:
   ```powershell
   mkdir C:\ProgramData\dossier
   # Download WinSW-x64.exe, rename to dossier-service.exe
   Move-Item WinSW-x64.exe C:\ProgramData\dossier\dossier-service.exe
   ```

2. **Copy the service config** from this directory:
   ```powershell
   Copy-Item dossier-service.xml C:\ProgramData\dossier\
   ```

3. **Run the install script** (PowerShell as admin):
   ```powershell
   .\install.ps1 -InstallDir C:\ProgramData\dossier
   ```
   The script creates a venv, installs dossier + regista, generates the
   environment file (prompts for DSN, project, key path), and installs +
   starts the service.

4. **Verify** the service is running:
   ```powershell
   Get-Service dossier
   # Or open http://localhost:8000 in a browser
   ```

## Configuration

The environment is set in `dossier-env.cmd` (generated by the install script).
Key variables:

| Variable | Description |
|---|---|
| `REGISTA_DSN` | Postgres connection string |
| `DOSSIER_PROJECT` | regista project/schema name |
| `REGISTA_KEY_PATH` | Path to HMAC keyset file |
| `DOSSIER_SESSION_SECRET` | Signed-cookie secret (auto-generated) |
| `DOSSIER_PASSWORD_SCRYPT_N` | scrypt N parameter — **set to 16384 on Windows** (Plan 016 spike: N=131072 fails with "memory limit exceeded" on Windows Python 3.14) |
| `DOSSIER_AUTH_BACKEND` | `local` or `ldap` |

## Service management

```powershell
# Stop
& "C:\ProgramData\dossier\dossier-service.exe" stop

# Start
& "C:\ProgramData\dossier\dossier-service.exe" start

# Restart
& "C:\ProgramData\dossier\dossier-service.exe" restart

# Uninstall
& "C:\ProgramData\dossier\dossier-service.exe" uninstall
```

The service auto-starts on boot (`startmode=Automatic` with a 30-second delay).

## Security notes

- No secrets are baked into the XML config or the image.
- The session secret is auto-generated at install time and stored in
  `dossier-env.cmd` (protected by NTFS ACLs on `C:\ProgramData`).
- The HMAC key file and users file are in `C:\ProgramData\dossier\secrets\`.
- For production, store secrets via the suite secret backend
  (`regista.secrets.resolve`) rather than plaintext files — see Plan 013 WI-4.1.

## Troubleshooting

- **Service won't start:** Check `$InstallDir\logs\dossier.out.log` and
  `dossier.err.log`.
- **scrypt memory error:** Ensure `DOSSIER_PASSWORD_SCRYPT_N=16384` in
  `dossier-env.cmd` (the install script sets this automatically).
- **Can't connect to Postgres:** Verify the DSN and network reachability from
  the service account (SYSTEM by default).
