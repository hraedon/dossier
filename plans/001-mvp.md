# Plan 001 — MVP: a usable tracker that is a provenance instrument

**Status:** Charter scaffold built 2026-06-20 (README, AGENTS, design spine
`docs/provenance-model.md`, draft workflow YAML, pyproject skeleton). No code yet;
no regista instance yet. (proposed 2026-06-20)
**Author:** Opus 4.8
**Strategic role:** Stand up the thinnest human front-end over regista that a team
can actually use, while turning on regista's built-in provenance (G1–G3 in the
design spine) so the activity log *is* a verified chain. Lightest = least bespoke
code, not least infra; dedicated Postgres/regista is fine.

## Ground truth at time of writing

- regista 0.4.0 is the backend (Postgres 15+; in-memory backend for tests only).
- The work model is regista's; dossier adds no state schema. See
  `docs/provenance-model.md` (the contract).
- The status model is fixed and declared in
  `src/dossier/workflows/dossier.workflow.yaml` (draft — not yet lint-validated).
- UI is server-rendered FastAPI + Jinja + patina, family style. No SPA.

## Principles this plan must hold

- **All work-state changes are regista events; dossier never owns writes** (AGENTS).
- **Actor binding is the root of provenance** — no anonymous/spoofed writes.
- **Built-in provenance on (HMAC + hash chain); Ed25519/timestamping/witness are
  deferred seams, not MVP.**
- **No work-domain identifiers committed.**

## Work items

- **WI-1 — Stand up regista for dossier.** Dedicated regista instance/schema +
  Postgres (docker-compose for dev; regista's `docker-compose.test.yml` is the
  reference). Install regista editable. Smoke-test create-work-item + transition +
  event replay against it.
- **WI-2 — Validate & load the workflow.** Run regista's workflow lint on
  `dossier.workflow.yaml`; fix to conform; register the workflow version. This is
  the design spine made real — do it before UI.
- **WI-3 — regista gateway.** A thin dossier module wrapping the regista facade
  (`work_items`, `events`, `transitions`) with the actor always injected. The only
  place dossier talks to regista; everything else goes through it. Enforces
  "actor on every mutation."
- **WI-4 — Auth + actor resolution.** Session auth (LDAP-pluggable, reuse the
  cert-watch pattern). The authenticated principal resolves to a regista
  `actor_kind=human` actor. This *is* provenance guarantee G1 — build it early.
- **WI-5 — Core UI: list/board + issue view.** Per-project list grouped by status,
  filterable by assignee; issue detail with create / edit / reassign / transition
  (transitions offered come from the workflow, enforced by regista). patina styling.
- **WI-6 — Comments.** Comment = a regista event on the work-item. Renders in the
  history.
- **WI-7 — The verified-history view (the differentiator).** Render the work-item's
  event log: per change — what, who (human/agent, on_behalf_of), when — plus an
  integrity status from a replay/verify call (chain intact / broken). This is G2+G3
  made legible and is the project's subject signature.
- **WI-8 — Issue display keys.** Mint `DOSSIER-<n>` keys over regista work-item ids
  (per-project sequence). Human-friendly, Jira-feel.
- **WI-9 — HTTP API under the UI.** The UI consumes a dossier API so an agent
  client can later use the same backend. No agent client in MVP — just don't build
  the UI as the only entry point.
- **WI-10 — `docs/publication-review.md`** — sanitization review gating any push.

## Sequencing

WI-1 → WI-2 (backend real + workflow valid) before any UI. WI-3 + WI-4 next (the
gateway and actor binding — the provenance foundation). Then WI-5/WI-6/WI-8 (the
usable tracker), then WI-7 (the verified-history view — the payoff). WI-9 rides
along with WI-5 (build the UI on the API). WI-10 before any remote.

## North star (explicitly NOT MVP)

Front the *same* work-items agents touch via agent-notes, so a single item shows a
mixed human+agent verified chain — the strongest demonstration of G1 and the answer
to the regulated-workplace blocker. The actor model is built to allow this as a
later config + workflow-alignment step. Also later: the multi-project federated
"pane-of-glass" view regista's spec envisions.
